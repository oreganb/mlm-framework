# C08 — Response Derivative Engine

## Purpose
Response Derivative Engine in the [[C-Series]]. Layer: **RL/Q-Learning**.

## Formula

### LaTeX
$$RD_i(t) = \frac{\partial \text{Flex}^{\uparrow}(t)}{\partial x_i(t)} = \lambda_i^*(t)$$

### Description
Response Derivative computed as shadow price λ*_i from C01 LP dual solution. Each input x_i (G_POA, SoC, T_out, price) has an associated RD measuring flexibility sensitivity.

### Method
LP dual prices, sensitivity analysis

## AI Context (Cursor Alignment)
RD is Brian's original theoretical contribution. λ*_i are the dual variables of the C01 LP — automatically computed by any LP solver (e.g., scipy.linprog, Gurobi). RD > 0 means increasing x_i would increase upward flexibility. RD = 0 means x_i is not currently binding. Essential for governance decisions in C10.
