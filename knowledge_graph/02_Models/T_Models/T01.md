# T01 — Building Thermal RC Model

## Purpose
Models the dynamic thermal response of a building zone using an RC (Resistor-Capacitor) electrical network analogy. Produces indoor temperature time series T_in(t) given all heat flows. This is the **foundational deterministic model** — its outputs constrain the feasible space for all HVAC and flexibility decisions.

## Layer
**Deterministic** — outputs are physics-based point values (no uncertainty). Uncertainty enters via stochastic inputs (T_out from W02, Q_occ from O-Series). Hard constraint: T_in must remain within [T_min, T_max] comfort band.

---

## Formula

### LaTeX (Single-Zone RC)
$$C \frac{dT_{in}}{dt} = \frac{T_{out} - T_{in}}{R_{env}} + Q_{solar} + Q_{int} + Q_{HVAC}$$

**Discrete-time (Euler, Δt timestep):**
$$T_{in}[k+1] = T_{in}[k] + \frac{\Delta t}{C} \left( \frac{T_{out}[k] - T_{in}[k]}{R_{env}} + Q_{solar}[k] + Q_{int}[k] + Q_{HVAC}[k] \right)$$

**Multi-zone extension:**
$$C_j \frac{dT_j}{dt} = \sum_{i \neq j} \frac{T_i - T_j}{R_{ij}} + \frac{T_{out} - T_j}{R_{env,j}} + Q_{solar,j} + Q_{int,j} + Q_{HVAC,j}$$

### Python Pseudocode
```python
def thermal_rc_step(T_in, T_out, Q_solar, Q_int, Q_HVAC,
                    R_env, C_zone, dt=900):
    """
    Single-zone RC thermal model — one timestep (default 15 min = 900s)

    Inputs (all weighted in formula):
      T_in    : Current indoor temperature (°C)  [from previous timestep or DS-SENSOR]
      T_out   : Outdoor temperature (°C)         [from W02, DS-WEATHER.temperature_outdoor]
      Q_solar : Solar heat gain (W)              [from W01+T04, weight=1.0]
      Q_int   : Internal heat gain (W)           [from O01+L04, weight=1.0]
      Q_HVAC  : HVAC thermal output (W)          [from C01 control signal, positive=heating]
      R_env   : Envelope thermal resistance (K/W)[from DS-BUILDING.thermal_resistance]
      C_zone  : Thermal capacitance (J/K)        [from DS-BUILDING.thermal_capacitance]
      dt      : Timestep (s), default 900 = 15min
    """
    dT = (dt / C_zone) * (
        (T_out - T_in) / R_env    # [w = 1/R_env, envelope conductance]
        + Q_solar                  # [w = 1.0, solar gain, no attenuation]
        + Q_int                    # [w = 1.0, internal gains]
        + Q_HVAC                   # [w = 1.0, HVAC output]
    )
    T_in_next = T_in + dT
    return T_in_next

def check_comfort_constraint(T_in, T_min=18.0, T_max=24.0):
    """Hard constraint — violation triggers C-Series alert"""
    return T_min <= T_in <= T_max
```

### Plain English
Think of the building as a leaky bucket of heat. The capacitance C tells you how much thermal mass the building has (concrete is high, lightweight timber is low). R_env is the insulation — the higher it is, the slower heat leaks out. At each 15-minute step, we calculate: heat leaking through the envelope + solar gain coming through windows + heat from people and equipment + heat from the HVAC. Divide by C and multiply by time to get how much the temperature changes.

## AI Context
- **Never hardcode R_env or C_zone** — always look up from DS-BUILDING by building_id and construction_year bracket
- **T_in[k+1] must always be checked against comfort bounds** before being returned — call `check_comfort_constraint()`
- **Q_solar must come from T04 (which calls W01)** — never use raw irradiance directly in T01
- **Timestep Δt = 900s** is fixed for ODEON deployment; make this a named constant `T01_DT_SECONDS = 900`
- **Multi-zone variant** (T12) must be called when building has >1 thermal zone — check `DS-BUILDING.zone_count`
