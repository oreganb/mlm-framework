# B01 — Battery SoC Prediction Model

## Purpose
Predicts battery State of Charge (SoC) at each timestep using a hybrid physics + ML approach. Physics-based Coulomb counting gives the deterministic trajectory; an LSTM correction layer accounts for non-linear aging, temperature effects, and measurement noise.

## Layer
**AI/ML** — hybrid model. The Coulomb counting step is deterministic; the LSTM correction is learned. Outputs SoC with uncertainty bounds.

---

## Formula

### LaTeX
**Coulomb counting (physics base):**
$$E_{t+1} = E_t + \eta_{ch} \cdot P_{ch}[t] \cdot \Delta t - \frac{P_{dis}[t]}{\eta_{dis}} \cdot \Delta t$$

$$SoC_{t+1} = \frac{E_{t+1}}{E_{nom}}$$

**LSTM correction:**
$$SoC_{corrected}[t] = SoC_{physics}[t] + f_{LSTM}(\mathbf{x}_{hist})$$

Where $\mathbf{x}_{hist}$ = [T_bat, P_ch/dis history, SoC history, cycle_count]

**Constraints:**
$$SoC_{min} \leq SoC[t] \leq SoC_{max}$$
$$0 \leq P_{ch}[t] \leq P_{ch,max}, \quad 0 \leq P_{dis}[t] \leq P_{dis,max}$$

### Python Pseudocode
```python
def battery_soc_step(E_t, P_ch, P_dis, eta_ch, eta_dis,
                     E_nom, SoC_min, SoC_max,
                     P_ch_max, P_dis_max, dt=900,
                     lstm_model=None, x_hist=None):
    """
    B01 Battery SoC Model — one timestep

    Inputs:
      E_t       : Current stored energy (Wh)   [state variable, from DS-BATTERY.init_soc or prev step]
      P_ch      : Charging power (W)            [from C01 dispatch, DS-BATTERY.max_charge_rate]
      P_dis     : Discharging power (W)         [from C01 dispatch]
      eta_ch    : Charge efficiency (fraction)  [w=0.95, from DS-BATTERY.charge_efficiency]
      eta_dis   : Discharge efficiency (frac)   [w=0.95, from DS-BATTERY.discharge_efficiency]
      E_nom     : Nominal capacity (Wh)         [from DS-BATTERY.capacity_wh]
      SoC_min   : Minimum SoC (fraction)        [w=0.10, from DS-BATTERY or policy]
      SoC_max   : Maximum SoC (fraction)        [w=0.90, from DS-BATTERY or policy]
      P_ch_max  : Max charging power (W)        [from DS-BATTERY.max_charge_rate]
      P_dis_max : Max discharge power (W)       [from DS-BATTERY.max_discharge_rate]
      dt        : Timestep (s), default 900
      lstm_model: Trained LSTM correction model (optional, from F-Series)
      x_hist    : History window for LSTM [T_bat, P_net, SoC] × last 24 steps
    """
    # Clamp to rated power limits [w = P_ch_max, P_dis_max]
    P_ch = min(P_ch, P_ch_max)
    P_dis = min(P_dis, P_dis_max)

    # Convert to Wh for this step (dt in seconds → hours)
    dt_h = dt / 3600.0

    # Coulomb counting [w_ch = eta_ch = 0.95, w_dis = 1/eta_dis = 1/0.95]
    E_next = E_t + eta_ch * P_ch * dt_h - (P_dis / eta_dis) * dt_h

    # Physics SoC
    SoC_physics = E_next / E_nom  # [w = 1/E_nom]

    # LSTM correction (if model available)
    if lstm_model is not None and x_hist is not None:
        delta_SoC = lstm_model.predict(x_hist)  # learned correction term
        SoC_corrected = SoC_physics + delta_SoC
    else:
        SoC_corrected = SoC_physics

    # Apply hard constraints [SoC_min=0.10, SoC_max=0.90]
    SoC_final = clip(SoC_corrected, SoC_min, SoC_max)
    E_final = SoC_final * E_nom

    return {
        "SoC": SoC_final,
        "E_stored_wh": E_final,
        "SoC_physics": SoC_physics,
        "SoC_correction": SoC_corrected - SoC_physics
    }
```

### Plain English
Start with how much energy is in the battery. Add whatever was charged (multiplied by charge efficiency, since some energy is lost as heat during charging). Subtract whatever was discharged (divided by discharge efficiency, since more energy must be drawn from the battery than reaches the load). Divide by total capacity to get SoC. The LSTM then applies a small learned correction based on recent history — accounting for factors like temperature-induced capacity reduction that the simple physics equation misses.

## AI Context
- `eta_ch` and `eta_dis` must be loaded from **DS-BATTERY by battery_id** — different chemistries have different efficiencies
- **Never allow SoC outside [SoC_min, SoC_max]** — apply hard clamp before returning; this is a physics + warranty constraint
- **P_ch and P_dis are mutually exclusive** per timestep — cannot charge and discharge simultaneously; enforce this in C01 as a binary constraint
- The **LSTM model weights** are updated by the F-Series federated learning cycle — load latest checkpoint from `F01.output.theta_global`
- For ODEON deployment: battery specs come from **HotDrop meters** and **Milesight AM319** data pipeline
