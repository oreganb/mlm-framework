# R01 — PV Generation Model

## Purpose
Computes AC power output of a photovoltaic array given plane-of-array irradiance and cell temperature. The canonical PV model for all ODEON buildings with solar panels.

## Layer
**Deterministic** — physics-based point calculation. Uncertainty enters from W01 (G_POA ensemble), making R01 the bridge between stochastic weather inputs and deterministic generation outputs.

---

## Formula

### LaTeX
$$P_{PV} = G_{POA} \cdot A \cdot \eta_{PV} \cdot [1 - \beta(T_{cell} - 25)] \cdot \eta_{inv}$$

**Cell temperature:**
$$T_{cell} = T_{out} + \frac{G_{POA}}{G_{NOCT}} \cdot (T_{NOCT} - T_{out,NOCT})$$

**Ensemble-aware version (using W01 quantiles):**
$$P_{PV}^{(q)} = G_{POA}^{(q)} \cdot A \cdot \eta_{PV} \cdot [1 - \beta(T_{cell} - 25)] \cdot \eta_{inv}, \quad q \in \{p10, p50, p90\}$$

### Python Pseudocode
```python
def compute_pv_power(G_POA_q, T_out, panel_params, inverter_params):
    """
    R01 PV Generation Model

    Inputs:
      G_POA_q      : Dict with keys p10, p50, p90, ensemble (from W01)
                     Source: W01.output, drawn from DS-NWP.irradiance_*
      T_out        : Outdoor temperature (°C) from W02, DS-WEATHER.temperature_outdoor
      panel_params : Dict from DS-PV:
                     - area_m2     : float, panel area (m²)   [w = multiplicative]
                     - eta_stc     : float, STC efficiency     [w = 0.18–0.22 typical]
                     - beta_coeff  : float, temp coefficient   [w = -0.0035 per °C typical]
                     - T_NOCT      : float, NOCT temp (°C)    [typically 45°C]
                     - G_NOCT      : float = 800 (W/m²)       [standard NOCT irradiance]
                     - T_out_NOCT  : float = 20 (°C)          [NOCT ambient]
      inverter_params: Dict from DS-INVERTER:
                     - eta_inv     : float, inverter efficiency [w = 0.96 typical]
    """
    results = {}
    for q in ['p10', 'p50', 'p90']:
        G = G_POA_q[q]  # W/m²

        # Cell temperature correction
        T_cell = T_out + (G / panel_params['G_NOCT']) * (
            panel_params['T_NOCT'] - panel_params['T_out_NOCT']
        )  # [w = G/800, scaling factor]

        # Temperature-corrected efficiency
        eta_eff = panel_params['eta_stc'] * (
            1.0 - panel_params['beta_coeff'] * (T_cell - 25.0)
        )  # [w_beta = -0.0035/°C]

        # DC power
        P_DC = G * panel_params['area_m2'] * eta_eff  # [w = area*eta_eff]

        # AC power (after inverter)
        P_AC = P_DC * inverter_params['eta_inv']  # [w = 0.96]

        results[q] = max(0.0, P_AC)  # clamp negative (night)

    return results  # {'p10': float, 'p50': float, 'p90': float}
```

### Plain English
Multiply the irradiance hitting the panel by the panel area and its efficiency (how well it converts light to electricity). Reduce the efficiency slightly because hot panels work less well (the beta coefficient, typically -0.35% per °C above 25°C). Then apply the inverter efficiency (typically 96%) to convert DC to AC. Do this three times — for the pessimistic (p10), median (p50), and optimistic (p90) irradiance forecasts.

## AI Context
- Always call **W01 first**, then W02, then R01 — the dependency chain is strict
- `G_POA_q` must be the **dict output of W01**, not a raw irradiance value
- **Panel parameters must be loaded from DS-PV by building_id** — never use default values in production ODEON deployment
- Return all three quantiles (p10, p50, p90) — downstream models select the appropriate quantile based on their risk posture
- **Clamp output to ≥ 0** — negative values (night) are physically impossible
- For ODEON Aran Islands: check `DS-PV.azimuth_deg` and `tilt_deg` per building — these vary significantly
